Вот дерево модулей репозитория:

<MODULE_TREE>
{module_tree}
</MODULE_TREE>

Вот список всех потенциальных основных компонентов модуля {module_name} (это нормально, что некоторые компоненты могут не являться существенными для модуля):
<POTENTIAL_CORE_COMPONENTS>
{potential_core_components}
</POTENTIAL_CORE_COMPONENTS>

Пожалуйста, сгруппируйте компоненты в группы так, чтобы каждая группа представляла собой набор тесно связанных компонентов, которые вместе образуют более мелкий модуль. НЕ включайте компоненты, которые не являются существенными для модуля.

Сначала обоснуйте группировку на основе предоставленного контекста, затем сгруппируйте компоненты и верните результат в следующем формате:
<GROUPED_COMPONENTS>
{{
    "module_name_1": {{
        "path": <path_to_the_module_1>, # путь к модулю может быть файлом или директорией
        "components": [
            <component_name_1>,
            <component_name_2>,
            ...
        ]
    }},
    "module_name_2": {{
        "path": <path_to_the_module_2>,
        "components": [
            <component_name_1>,
            <component_name_2>,
            ...
        ]
    }},
    ...
}}
</GROUPED_COMPONENTS>
